## 第四轮修复（真正根因）：inset box-shadow 替代 border
- 用户反馈"单击卡片时上边框还是没显示"——这次锁定的是**普通卡片 hover 态**，不是编辑态
- **真正根因**：`.card-wrapper{overflow:hidden}`（为左滑删除裁切而设）裁切了**所有卡片**的 1px border 顶部，不只是编辑态。前几轮只改编辑态，没动普通卡片 border，所以问题一直在
- **方案**：把 `.card` 的 `border:1px solid` 改为 `box-shadow:inset 0 0 0 1px`（内阴影模拟边框）
  - inset 阴影画在卡片 padding-box 内部，永远在 `overflow:hidden` 的裁切范围之内，不可能被切掉
  - hover / editing / swiping-deleting 各状态都用 inset + 外层 glow 组合
- 完整改动（styles.css）：
  ```css
  .card{border:none;box-shadow:inset 0 0 0 1px var(--card-border)}
  .card:hover{box-shadow:inset 0 0 0 1px var(--accent),0 2px 12px rgba(92,64,51,0.08)}
  .card.editing{box-shadow:inset 0 0 0 1px var(--accent),<glow>}
  ```
- 关键教训：当父元素有 `overflow:hidden` 时，子元素的 border 会被边缘裁切；用 `inset box-shadow` 模拟边框是规避此问题的标准手法
- 注意：`.new-card` 的 dashed 边框无法用 inset 模拟（inset 不能画虚线），但 new-card 不被 .card-wrapper 包裹，无此问题
- 已部署到 Cloud Studio

## 金色边框顶部缺失 — 第二次修复
- **根因1**：`.card-wrapper` 有 `overflow:hidden`，编辑卡片的 `box-shadow:0 0 0 1px` 金色环和 1px border 在顶部被亚像素裁切
- **根因2**：夜间模式 `[data-theme="night"] .card.editing{box-shadow:0 0 16px var(--accent-glow)}` 完全覆盖了日间模式的 `0 0 0 1px var(--accent)` 1px环，导致夜间编辑卡片只有模糊光晕没有清晰边框
- **修复**：
  1. 新增 `.card-wrapper:has(.card.editing){overflow:visible}` — 编辑时取消 wrapper 的 overflow 裁切
  2. 夜间模式改为 `box-shadow:0 0 0 1px var(--accent),0 0 16px var(--accent-glow)` — 1px环 + 光晕组合
  3. `.new-card` 加 `box-shadow:0 0 0 1px var(--accent)` 作为实线环备份，与 dashed border 叠加确保四面可见
- `:has()` 选择器在现代浏览器（Chrome 105+/Safari 15.4+/Firefox 121+）已支持，2026年使用无问题

## 第三轮修复：改用柔和外发光替代边框（彻底解决顶部缺失）
- 前两轮 border 修复无效后，用户同意放弃边框方案，改用 box-shadow 柔和外发光
- 关键改动（styles.css + js/ui.js）：
  1. 删除 `.card-wrapper:has(.card.editing){overflow:visible}`（`:has()` 移动端 WebView 不可靠）
  2. 新增 `.card-wrapper.editing-wrapper{overflow:visible}`，由 JS 在 render() 给编辑卡片的 wrapper 加 `editing-wrapper` class
  3. `.card.editing` 不再改 border 色，改用 glow：日间 `0 2px 16px rgba(92,64,51,0.18)`，夜间 `0 0 24px rgba(212,165,116,0.30)`，均保留 `0 0 0 1px var(--card-border)` 维持轮廓
  4. `.new-card` dashed 边框从 accent 金色改为中性 card-border 色，金色指示交给 glow
- 教训：移动端 WebView 对 `:has()` 支持参差，需要可靠触发时用 JS 加 class 而非 `:has()`；glow 无明确边缘，即使有微量裁切也不会"缺一边"，容错性远高于 border
- 已部署到 Cloud Studio

## 第四轮修复（真正根因）：inset box-shadow 替代 border
- 用户反馈"单击卡片时上边框还是没显示"——这次锁定的是**普通卡片 hover 态**，不是编辑态
- **真正根因**：`.card-wrapper{overflow:hidden}`（为左滑删除裁切而设）裁切了**所有卡片**的 1px border 顶部，不只是编辑态。前几轮只改编辑态，没动普通卡片 border，所以问题一直在
- **方案**：把 `.card` 的 `border:1px solid` 改为 `box-shadow:inset 0 0 0 1px`（内阴影模拟边框）

  - hover / editing / swiping-deleting 各状态都用 inset + 外层 glow 组合
- 完整改动（styles.css）：
  ```css
  .card{border:none;box-shadow:inset 0 0 0 1px var(--card-border)}
  .card:hover{box-shadow:inset 0 0 0 1px var(--accent),0 2px 12px rgba(92,64,51,0.08)}
  .card.editing{box-shadow:inset 0 0 0 1px var(--accent),<glow>}
  ```
- 关键教训：当父元素有 `overflow:hidden` 时，子元素的 border 会被边缘裁切；用 `inset box-shadow` 模拟边框是规避此问题的标准手法

## 第四轮修复（真正根因）：inset box-shadow 替代 border
- 用户反馈"单击卡片时上边框还是没显示"——这次锁定的是**普通卡片 hover 态**，不是编辑态
- **真正根因**：`.card-wrapper{overflow:hidden}`（为左滑删除裁切而设）裁切了**所有卡片**的 1px border 顶部，不只是编辑态。前几轮只改编辑态，没动普通卡片 border，所以问题一直在
- **方案**：把 `.card` 的 `border:1px solid` 改为 `box-shadow:inset 0 0 0 1px`（内阴影模拟边框）
  - inset 阴影画在卡片 padding-box 内部，永远在 `overflow:hidden` 的裁切范围之内，不可能被切掉
  - hover / editing / swiping-deleting 各状态都用 inset + 外层 glow 组合
- 完整改动（styles.css）：
  ```css
  .card{border:none;box-shadow:inset 0 0 0 1px var(--card-border)}
  .card:hover{box-shadow:inset 0 0 0 1px var(--accent),0 2px 12px rgba(92,64,51,0.08)}
  .card.editing{box-shadow:inset 0 0 0 1px var(--accent),<glow>}
  ```
- 关键教训：当父元素有 `overflow:hidden` 时，子元素的 border 会被边缘裁切；用 `inset box-shadow` 模拟边框是规避此问题的标准手法
- 注意：`.new-card` 的 dashed 边框无法用 inset 模拟（inset 不能画虚线），但 new-card 不被 .card-wrapper 包裹，无此问题
